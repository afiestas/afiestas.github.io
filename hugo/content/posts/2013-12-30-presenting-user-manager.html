We are happy to announce a new small project called User Manager that we hope will make less scary the administration of system users when running KDE Plasma desktop.

Background:
A few months back in one of the sessions of the last UDS I attended the Kubuntu team (me included) wondered if we could replace the old KUser with something more "user friendly" since doing the most basic task with it was difficult even for the attendees of that session. After looking at all alternatives we realized that all existing KDE-friendly solutions were thought for sysadmins rather than for the average user. So the only option we had left was creating our own new thing.

Design:
When designing User Manager we used the personas we created for Solid and focused on Penny (as we do for Solid).

First thing we realized is that Penny (the average user) does not care about Groups. In modern user-oriented distributions there is no need for having to mess with Groups, so let's not even try to make the concept of group management user friendly.

Then we realized that from Users, not everything had to be shown, things like: system users (httpd, nobody, ftp, lightdm, rkit...), home folder or login shell are details of no interest for Penny, so we decided to just skip them in the new interface.

Also we realized that we have some functionality duplication with "kcm_useraccount", we decided that we should merge most of that functionality on User Manager.

Finally, we wanted to avoid popups as much as possible, so we decided to have a list of users on the left that allows you to show/edit information on the right.
